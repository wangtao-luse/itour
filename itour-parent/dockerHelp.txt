项目模块容器Docker命令
docker [options] IMAGE [COMMAND]
-d:创建一个守护式容器在后台一直运行
-i:表示运行容器;
-t:表示容器启动后会进入命令行,加上这个参数后，容器创建就能登录进去，即分配了一个伪终端.
--name:指定容器名字,后续可以通过名字进行容器管理
-p:端口映射
-v:给容器挂载存储卷，挂载到容器的某个目录 宿主机目录：容器目录 (将宿主机的目录挂载到容器中)
--restart=always:重启docker时，自动启动相关容器

Nginx 
1.创建挂载目录;
  mkdir -p ./nginx/{html,log,conf.d}
2.运行nginx
  docker run -d --name nginx -p 80:80 nginx
3.将nginx容器内部配置文件挂载到主机
  a.复制容器配置文件
   docker cp c10d0b833743:/etc/nginx/nginx.conf  ./
   docker cp c10d0b833743:/etc/nginx/conf.d/default.conf ./conf.d/
4.移除运行的nginx
 docker rm c10d0b833743
5.运行nginx 
docker run -d --name test-nginx -p 8082:80 -v /etc/nginx/nginx.conf:/etc/nginx/nginx.conf -v /etc/nginx/log:/var/log/nginx -v /etc/nginx/html:/usr/share/nginx/html -v /etc/nginx/conf.d:/etc/nginx/conf.d  --privileged=true nginx
